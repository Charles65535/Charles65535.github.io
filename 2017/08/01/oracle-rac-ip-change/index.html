Oracle RAC IP修改

发表于 2017-08-01 | 分类于 运维

一、修改前的IP地址

* #more /etc/hosts
* 127.0.0.1     localhost localhost.localdomain localhost4 localhost4.localdomain
* ::1           localhost localhost.localdomain localhost6 localhost6.localdomain
* 
* # Public
* 130.1.111.91     qarac1.localdomain     qarac1
* 130.1.111.92     qarac2.localdomain     qarac2
* 
* #Private
* 192.168.254.23     qarac1-priv.localdomain qarac1-priv
* 192.168.254.24     qarac2-priv.localdomain qarac2-priv
* 
* #Virtual
* 130.1.111.93     qarac1-vip.localdomain     qarac1-vip
* 130.1.111.94     qarac2-vip.localdomain     qarac2-vip
* 
* # SCAN
* 130.1.111.95     qarac-scan.localdomain     qarac-scan

二、停服务

在节点1上：

* #srvctl disable listener -n qarac1
* #srvctl disable listener -n qarac2

查看有哪些db：

* 实例名:$ORACLE_HOME/dbs/initSID.ORA 这里可以看到本机的数据库实例名
* 数据库名：$ORACLE_BASE/admin/SID/ 这个SID目录一般是数据库名字

停掉db

* #srvctl disable database -d ebank -n qarac1
* #srvctl disable database -d ebank -n qarac2
* #srvctl disable database -d esbdb -n qarac1
* #srvctl disable database -d esbdb -n qarac1
* 
* #srvctl disable vip -i 'qarac1-vip.localdomain'
* #srvctl disable vip -i 'qarac2-vip.localdomain'
* #srvctl disable scan_listener
* #srvctl disavle scan
* #srvctl stop listener -n qarac1
* #srvctl stop listener -n qarac2
* #srvctl stop database -d ebank -o immediate
* #srvctl stop database -d esbdb -o immediate
* #srvctl stop vip -n qarac1
* #srvctl stop vip -n qarac2
* #srvctl stop scan_listener
* #srvctl stop scan
* #crsctl stop crs

在两个节点上运行
ps -ef |grep grid
没有任何输出

（此处遇到一个问题：节点1顺利停掉，节点2还是有很多进程。 在节点2上执行crsctl stop has）

三、修改IP

修改两台hosts，修改两台IP地址，登陆到新地址。
先在node1执行：

* #crsctl start crs

立即在node2执行

* #crsctl start crs

查看当前状态

* #oifcfg getif
* 
* eth0 130.1.111.0 global public
* eth1 192.168.254.0 global cluster_interconnect

修改public IP

* #oifcfg delif -global eth0
* #oifcfg setif -global eth0/136.1.11.0:public
* 再次查看
* #oifcfg getif
* eth1 192.168.254.0 global cluster_interconnect
* eth0 130.1.11.0 global public

修改vip

* 查看节点1的vip
* #srvctl config vip -n qarac1
* VIP exists: /qarac1-vip/130.1.111.93/130.1.111.0/255.255.255.0/eth0,hosting node qarac1
* 
* 修改节点1的vip
* #srvctl modify nodeapps -n qarac1 -A 136.1.11.93/255.255.255.0/eth0
* 
* 再次查看节点1的vip
* #srvctl config vip -n qarac1
* VIP exists: /qarac1-vip/136.1.11.93/136.1.11.0/255.255.255.0/eth0,hosting node qarac1
* 
* 查看节点2的vip
* #srvctl config vip -n qarac2
* VIP exists: /qarac2-vip/130.1.111.94/130.1.111.0/255.255.255.0/eth0,hosting node qarac2
* 
* 修改节点2的vip
* #srvctl modify nodeapps -n qarac2 -A 136.1.11.94/255.255.255.0/eth0
* 
* 再次查看节点2的vip
* #srvctl config vip -n qarac2
* VIP exists: /qarac2-vip/136.1.11.94/136.1.11.0/255.255.255.0/eth0,hosting node qarac2

命令解释：

* Usage: srvctl modify nodeapps -n <node_name>[-o <oracle_home>] [-A <new_vip_address>]

备注：这里遇到一个问题：节点1修改vip的是输错了IP，莫名查看节点1的vip出现这的结果：

* #srvctl config vip -n qarac1
* VIP exists: /136.1.11.93/136.1.11.93/136.1.11.0/255.255.255.0/eth0,hosting node qarac1
* 而节点2正常：
* 
* VIP exists: /qarac2-vip/136.1.11.94/136.1.11.0/255.255.255.0/eth0,hosting node qarac2
* 最后执行 
* #srvctl modify nodeapps -n qarac1 -A qarac1-vip/255.255.255.0/eth0
* 然后再修改VIP，解决了这个奇怪的问题。

修改SCAN IP

* 查看SCAN IP
* #srvctl config scan
* SCAN name: qarac-scan, Network: 1/130.1.111.0/255.255.255.0/eth0
* SCAN VIP name: scan1,IP: /qarac-scan/130.1.111.95
* 
* 修改scan IP：
* #srvctl modify scan -n qarac-scan
* 
* 再次查看
* #srvctl config scan
* SCAN name: qarac-scan, Network: 1/136.1.11.0/255.255.255.0/eth0
* SCAN VIP name: scan1, IP: /qarac-scan/136.1.11.95

四、启服务

crs_stat -t 直到两个节点都显示正常

* srvctl enable listener -n qarac1
* srvctl enbale listener -n qarac2
* srvctl enable database -d ebank -n qarac1
* srvctl enable database -d ebank -n qarac2
* srvctl enable database -d esbdb -n qarac1
* srvctl enable database -d esbdb -n qarac1
* srvctl enable vip -i 'qarac1-vip.localdomain'
* srvctl enable vip -i 'qarac2-vip.localdomain'
* srvctl enable vip scan_listener
* srvctl enbale scan
* srvctl start listener -n qarac1
* srvctl start listener -n qarac2
* srvctl start scan_listener
* srvctl start database -d ebank
* srvctl start database -d esbdk
